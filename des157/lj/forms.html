Best Practices for Form Design

January 21, 2021

Forms have gotten a lot of attention by designers in recent years. We’ve learned some best practices for creating forms on the web that increase the likelihood of follow-through and make the web accessible to all. Salim Ansari shares 20 tips to boost form usability in a Medium article. I have read about best practices for form design in the past, so I’ll highlight some new things I learned. First, I have been wondering why asterisks (“*”) were slowly going away in forms, and it turns out the reason is because they’re bad design. While it’s good design to distinguish optional and required fields, asterisks are perceived as a mandatory action, and hey, who wants to feel controlled? A better way to distinguish between optional and required fields is to simply include an optional tag on the items that are optional.

Another thing that surprised me was that placeholder text is now a no-no. The reasoning behind this is because users lose the hint of placeholder text the second they click in an input field. A better practice is to include the hint underneath the input field so that the user can reference this if needed. I also never thought about input field sizes, but now I know that sizing fields accordingly is a deliberate design. For example email fields are longer than zip code fields. This is what designers would call an “affordance” because the user gets an extra hint about what to fill into a certain input field (making the user experience that much smoother).

One final note. I’m currently taking a writing class on user experience documentation and a big topic of discussion is the importance of accessibility. Tim Berner’s Lee wanted the web to be accessible to everyone—so we must design in this manner. I think it’s so cool that there’s guidelines and resources we can use as designers to create accessible web experiences. I had no idea about the minimum contrast ratio provided by WCAG, but now that I know, I can implement that into my design.
